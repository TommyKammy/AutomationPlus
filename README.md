# AutomationPlus
Meta-orchestration layer for codex-supervisor: uses n8n and Codex to automate epic evaluations, follow-up issue drafting, loop operations, roadmap planning, and Obsidian sync for trusted solo issue-driven development.

## Loop Observation
`csctl loop-status` is the read-only observation boundary for the AutomationPlus loop. The launcher/service contract it depends on is documented in [docs/launcher-service-contract.md](docs/launcher-service-contract.md).
